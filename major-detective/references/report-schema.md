# Report Schema

## Full Report

Use this structure for a专业真相报告:

1. 专业一句话画像
2. 用户真实问题复述
3. 信息完整度与需核验信息
4. 这个专业到底学什么
5. 常见误解
6. 适合人群
7. 不太适合的情况
8. 升学与就业路径
9. 风险雷达
10. 替代专业与相邻方向
11. 家长沟通建议
12. 官方核验清单
13. 总结：不是结论，而是决策条件

## Risk Radar Dimensions

Score with low / medium / high rather than precise numbers unless a visual output requires numbers.

- 本科就业压力
- 读研依赖度
- 学习周期
- 数学/代码压力
- 背诵/考试压力
- 实验/实践压力
- 城市与行业限制
- 转行友好度
- 家庭经济和时间成本
- 信息误解风险


## School-Major Candidate Pool

Use this structure when the user asks for university-major matching:

1. 用户输入与缺失数据
2. 数据边界：不是录取预测
3. 数据源与年份
4. 冲/稳/保候选池表格
5. 院校-专业质量信号
6. 用户适配理由
7. 主要风险
8. 必须核验的问题
9. 位次/招生计划更新后的复查动作

Candidate table:

| 档位 | 院校 | 专业/专业类 | 城市 | 往年最低位次/分数 | 计划变化 | 专业实力信号 | 适配理由 | 主要风险 | 下一步核验 |
|---|---|---|---|---|---|---|---|---|---|

## Parent-Facing Template

```text
给家长看的冷静版：

这个专业可以关注，但不应只看专业名字。需要重点核验三件事：
1. 学校培养方案：核心课程和实践安排是否与孩子能力匹配。
2. 出路结构：本科就业、读研、考公考编分别是什么路径。
3. 兜底方案：如果孩子读到大二发现不适合，是否有转专业、辅修、考研跨方向、实习转向的空间。
```
